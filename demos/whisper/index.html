Whisper demo - overview

In this scenario, the agent is called by a customer and they can talk to each other. The manager can connect to the agent and hear both the agent and the customer. The manager can also "whisper" to the agent, but can not be heard by the customer.

When the agent starts taking calls, the page first requests access to the microphone and then registers for incoming calls. Upon answering a customer call the previously obtained media stream, with audio from the microphone, is used as the local stream. Manager calls gets passed the output from an WebAudio context that mixes the microphone stream and the stream of received audio from the customer call.

The customer and manager pages just make normal WebRTC calls to the agent.

In each page, a token is needed. The "Get token" button activates a dialog that uses the webrtc_generate_token Web Service API to generate a token. The parameters passed are displayed in the dialog. You need to enter your account details to allow the dialog to call the web service.
